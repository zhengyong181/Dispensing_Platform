# tests 目录规则

测试目录跟随当前已经创建的项目和真实行为，不提前铺满空测试项目。

## 允许

- 为已实现的契约、算法、状态机、配置加载、持久化和安全流程创建测试。
- 使用 Simulator 验证设备相关流程。
- 使用架构测试检查依赖方向、Prism 边界和客户隔离规则。
- 为 schema、迁移、Trace、IR hash 和恢复流程补充回归测试。

## 禁止

- 不得创建大量空测试项目或只为覆盖率存在的假测试。
- 默认测试不得依赖真机、PLC、相机、现场网络或客户私有资源。
- 不得通过放宽断言掩盖架构边界破坏。
- 不得让测试直接依赖不应公开的跨层实现，除非该层明确暴露了测试友元边界。

## 交互方式

- 修改哪个项目或逻辑区域，就优先运行对应测试。
- 依赖方向变化时运行架构测试。
- 安全相关逻辑变化时必须补充失败路径和恢复路径测试。
