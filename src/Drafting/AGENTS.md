# Drafting 层规则

Drafting 负责编辑器领域模型、几何、命令、吸附、图层、块、导入导出和工艺标注。它表达用户设计意图，不直接驱动设备。

## 允许

- 定义图纸文档、图元、几何算法、编辑命令、撤销重做和工艺标注。
- 通过 Process / IR 语义生成可编译的工艺数据。
- 在 UI 适配边界使用 WPF 渲染，但核心模型和算法应保持可测试。

## 禁止

- 不得依赖 HAL、PLC SDK、相机 SDK、运动控制 SDK 或具体控制器协议。
- 不得直接执行运动、点胶、IO 或真机预览。
- 不得把 UI 交互状态混进不可变工艺模型。
- 不得绕过 Process 直接生成可下发生产程序。

## 交互方式

- Drafting 输出工艺语义和 IR 输入，不直接调用设备。
- 与 UI 的交互通过命令、ViewModel 或模块适配层完成。
- 与执行链路的交互通过 Process 编译、仿真和派生产物状态完成。
- Drafting 数据结构变化必须同步 `docs/4 Drafting-Subsystem.md`。
