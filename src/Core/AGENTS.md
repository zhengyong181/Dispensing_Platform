# Core 层规则

Core 是稳定契约、基础类型和横切抽象所在层。它应当是最少依赖、最稳定的部分。

## 允许

- 定义跨层接口、结果类型、错误码、事件契约、配置抽象、日志抽象和基础值对象。
- 定义不依赖 UI 和硬件 SDK 的通用服务契约。
- 使用 UnitsNet 表达物理量和工程量。

## 禁止

- 不得引用 WPF、Prism、UI 控件库或 DesignSystem 资源。
- 不得引用具体 HAL 实现、厂家 SDK、PLC SDK 或相机 SDK。
- 不得包含客户专属逻辑、具体业务流程或设备执行细节。
- 不得随意改变公共契约；破坏性变更需要同步文档并评估 ADR。

## 交互方式

- Core 只提供契约和基础抽象，不主动调用外层。
- 外层通过依赖注入绑定 Core 契约的实现。
- 公共契约变更必须同步 `docs/3 Core-Contracts.md`。
